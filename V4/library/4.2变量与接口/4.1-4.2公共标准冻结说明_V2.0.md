# 4.1—4.2公共标准冻结说明 V2.0

> 冻结日期：2026-07-20  
> Schema：`BLUE_HUB_CH4_SCHEMA_V2`  
> Schema版本：`2.0.0`  
> 公共案例：`common_case_v2`  
> 状态：`FROZEN_INTERFACE`

## 1. 冻结范围

以下内容自V2.0起不再由4.3—4.9修改：

- 系统空间边界与结算终点；
- 公共测点名称；
- 优化层单位和维度顺序；
- 非负端口幅值及端口方向；
- `requested → accepted → actual`链路；
- `BOUNDARY → REQUEST → RESPONSE → LEDGER → STATE_COMMIT → EVALUATION`执行顺序；
- 损耗、辅机、状态、交付与评价的唯一所有者；
- 公共数据包顶层结构和字段含义；
- 4.4仅用实际值记账且禁止隐式闭合残差；
- 4.5拥有SOC和氢库存，4.6拥有算力队列，4.7拥有交付通道；
- 电力送端进入4.4、受端用于结算；算力服务统一以MWh-CS交付。

工程参数、场址、容量、价格、效率和初态可通过`parameterVersion`更新，不构成Schema变更。

## 2. 强制使用方式

1. 联合运行必须显式调用`common_config_4_2`；
2. 配置必须通过`validate_common_schema_4_2(cfg,true)`；
3. 交换包必须由`common_packet_4_2`创建；
4. 可控电气端口必须由`common_dispatch_port_4_2`创建或形成完全相同的字段；
5. 4.3—4.7数据包必须通过`validate_module_packet_4_2`；
6. 历史模块不符合本版时修改适配器或历史模块，不回改公共V2合同；
7. `engineering_base`在P0参数和项目签认完成前必须被阻断。
