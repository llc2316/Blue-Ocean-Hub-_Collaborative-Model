# 4.2 第四章全量变量与接口字典（冻结版）

> 版本：V2.0  
> 状态：**FROZEN_INTERFACE**  
> Schema：`BLUE_HUB_CH4_SCHEMA_V2`  
> Case：`common_case_v2`  
> 配套文件：4.1 V2.0冻结口径、`common_config_4_2.m`、`validate_common_schema_4_2.m`、`validate_module_packet_4_2.m`  
> 机器可读实现：`common_config_4_2.m`、`common_packet_4_2.m`、`common_dispatch_port_4_2.m`、`validate_common_schema_4_2.m`、`validate_module_packet_4_2.m`  
> 统一优化层单位：MW、MWh、h、kg、kg/h、CNY/MWh、CNY/kg、kgCO₂e。

## 4.2.1 命名与使用规则

1. MATLAB公共配置采用`lowerCamelCase`；逐时功率变量以`MW`结尾，电量以`MWh`结尾，氢量以`Kg`结尾。
2. 设备模型允许使用W/var/s，但必须经4.3适配器转为MW/h后接入联合模型。
3. 可控端口使用`Requested`、`Accepted`、`Actual`后缀，不以一个`power`字段同时表达三种含义。
4. 优化层功率端口采用非负幅值；只有派生的`pBessNetMW=pBessDischargeMW-pBessChargeMW`允许有符号。
5. 所有逐时字段共享`timeH`、`caseId`、`scenarioId`、`meterPoint`和`qualityFlag`。
6. 生产模块不得修改其他模块拥有的字段；4.4只读取`Actual`字段形成总账。
7. 公共结构的唯一机器可读来源是`common_config_4_2.m`；本文件用于解释字段含义，不建立第二套代码定义。
8. 联合运行不得使用模块默认参数静默补齐缺失公共字段；字段缺失、版本不符、单位或测点不符必须停止运行。
9. 公共Schema结构冻结，工程数值只通过`parameterVersion`更新；后续4.3—4.9出现冲突时修改适配器，不回改V2字段含义。
10. 模块交换必须使用`meta/axis/ports/state/product/service/loss/quality/audit`九个顶层区块。

### 4.2.2.1 维度冻结

| 数据类型 | 固定维度 | 说明 |
|---|---|---|
| 单端口逐时序列 | N×1 | N为时间点数 |
| 多源逐时序列 | N×S | S为源/设备数量 |
| 聚合联合场景 | N×K | K为场景数量 |
| 多源联合场景 | N×S×K | 不得交换为S×N×K |
| 场景概率 | 1×K | 非负且求和为1 |
| 时间轴 | N×1 | 严格递增，表示区间起点 |

功率量表示区间平均功率；电量、产氢量和服务量表示区间累计量；SOC、氢库存、任务队列和设备状态表示区间末状态。状态不得线性插值。

### 4.2.2.2 公共可控端口

每个可控电气端口统一包含：

```text
requestedMW / acceptedMW / actualMW
minMW / maxMW
upCapabilityMW / downCapabilityMW
constraintCode / operatingState
meterPoint / qualityFlag
```

所有字段均为非负幅值。端口方向由端口名称和4.1责任表确定，禁止通过正负号重新解释。`BOUNDARY`阶段必须给出有限的上下界和调节能力，`REQUEST`阶段必须给出请求值，`RESPONSE`阶段必须给出有限的请求、接受和实际值。

公共母线核算式为项目统一接口定义：

\[
$r_t^P=P_t^{src}+P_t^{BESS,dis}+P_t^{other}+P_t^{imp} -P_t^{BESS,ch}-P_t^{ely}-P_t^{DC}-P_t^{export}-P_t^{marine}-P_t^{src,aux}-P_t^{common,aux}-P_t^{post}-P_t^{spill}.$
\]

其中各项必须为对应测点的`Actual`值；该总账式不替代各模块内部状态方程。

## 4.2.9 最小交换数据包

每个模块必须通过`common_packet_4_2`创建数据包，不得自行拼装另一种顶层结构：

```text
meta: schemaId, schemaVersion, caseId, caseMode, parameterVersion, moduleId, phase
axis: timeH, dtH, scenarioId, scenarioProbability
ports: 一个或多个公共可控端口
state: 模块拥有的时段末状态
product: 氢等产品累计量及库存接口
service: 算力、电力受端和海洋服务交付量
loss: 模块唯一负责的损耗与辅机
quality: qualityFlag, dataSourceType, calibrationVersion
audit: assumptionNotice, createdBy, implicitDefaultsUsed
```

标准端口名称固定为：

| 模块 | `ports`下允许的正式端口 | 测点 |
|---|---|---|
| 4.3 | `source` | `MP-03-SOURCE-POI` |
| 4.5 | `bessCharge`、`bessDischarge` | `MP-04-COMMON-BUS` |
| 4.5 | `electrolyzer` | `MP-ELECTROLYZER` |
| 4.6 | `dcFacility` | `MP-DC-FACILITY` |
| 4.7 | `exportSend`、`gridImport` | `MP-CABLE-SEND` |
| 4.7 | `marine` | `MP-04-COMMON-BUS` |

4.4不建立设备端口，只读取上述`RESPONSE`数据包的`actualMW`形成`LEDGER`数据包。模块专属状态可以追加，但不得覆盖公共字段含义，也不得修改其他模块状态。

## 4.2.10 校验与发布规则

1. `common_config_4_2('interface_smoke')`必须通过严格校验；其结果必须携带假设声明。
2. `common_config_4_2('engineering_base')`在P0字段未补齐和未签认时必须校验失败。
3. 联合运行前调用`validate_common_schema_4_2(cfg,true)`；不得捕获错误后继续使用模块默认值。
4. 4.3—4.7每次交换前调用`validate_module_packet_4_2(packet,moduleId,cfg,true)`。
5. `schemaId`、`schemaVersion`、字段含义、维度顺序、单位、测点、端口方向和所有权冻结，不因参数更新而改变。
6. 参数更新时只升级`parameterVersion`，保存输入快照、证据矩阵和校验报告。
7. 不允许4.4以隐式弃电、缺供或储能响应自动闭合残差；相关量必须由4.9显式给出，4.4只核验。
8. 4.4闭合、损耗单次计量、初末状态、收益交付点和方案同口径必须纳入回归测试。
9. `run_all_tests_4_2`必须全部通过后，后续模块才允许接入。
10. 发布时保存4.1、4.2机器可读文件的SHA-256清单；发现漂移即阻止联合运行。

## 4.2.11 冻结规则

本版公共合同冻结后，后续小节不得要求回改V2以迁就历史代码。历史代码应通过适配器完成字段、单位和测点转换。只有出现物理错误、法规冲突或必要端口无法表达时，才可建立新Schema；新Schema必须使用新编号、保留V2文件并提供逐字段迁移表。
