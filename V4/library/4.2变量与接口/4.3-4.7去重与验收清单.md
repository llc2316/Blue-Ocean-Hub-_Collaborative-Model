# 4.3—4.7模块去重、对接与验收清单

> 版本：V2.0  
> 上位标准：`4.1系统边界与统一建模口径_V2.0_冻结版.md`、`4.2第四章全量变量与接口字典_V2.0_冻结版.md`  
> Schema：`BLUE_HUB_CH4_SCHEMA_V2` / `2.0.0`  
> 公共案例：`common_case_v2`  
> 用途：指导4.3—4.7历史模型迁移、模块联调、职责去重和最终验收  
> 数据原则：没有工程或企业依据的数值必须标注 **[假设值，待企业调研校准]**；尚未核验文献来源的专业关系必须标注 **[需查证文献支撑]**。

## 1. 本清单的约束力

4.1冻结系统边界、测点、时间尺度、功率方向、损耗归属和模块责任；4.2将其固化为机器可读配置、数据包和校验器。4.3—4.7不得为迁就历史代码修改4.1—4.2，也不得另建第二套公共变量或总功率平衡。

发生冲突时执行优先级为：

```text
4.1冻结边界与责任
    > 4.2机器可读Schema与校验器
    > 4.3—4.7专业模型内部定义
    > 演示脚本和模块默认值
```

历史模型与V2不一致时，应修改历史模型或增加适配器。联合运行不得捕获Schema错误后继续计算，不得用模块默认值静默补齐公共参数。

## 2. 4.3—4.7总体关系

4.3—4.7不是依次单向执行的流水线。正确链路为“发布边界—统一调度—专业响应—实际总账—状态提交”：

```text
4.1—4.2：公共配置、时间轴、场景、测点、单位、参数版本
                          ↓
        ┌───────── BOUNDARY ─────────┐
        │ 4.3 源侧可行域            │
        │ 4.5 电池/制氢/储氢可行域   │
        │ 4.6 算力负荷可行域         │
        │ 4.7 外送/物流/用能可行域   │
        └────────────┬───────────────┘
                     ↓
            4.9或接口联调规则器
              产生REQUEST指令
                     ↓
        4.3/4.5/4.6/4.7执行RESPONSE
        返回accepted、actual和约束原因
                     ↓
             4.4读取全部actual
             执行公共母线LEDGER
                     ↓
        各状态所有者执行STATE_COMMIT
                     ↓
                 4.8评价
```

固定执行阶段如下，后续模块不得更改顺序：

```text
BOUNDARY → REQUEST → RESPONSE → LEDGER → STATE_COMMIT → EVALUATION
```

在4.9正式完成前，可以使用明确标注为 **[假设值，待调度策略校准]** 的规则调度器产生REQUEST，但规则调度器不得进入4.3—4.7专业模块内部。

## 4. 与4.1、4.2的统一对接要求

### 4.1 公共配置

所有联合入口首先执行：

```matlab
cfg = common_config_4_2('interface_smoke');
validate_common_schema_4_2(cfg,true);
```

工程计算使用`engineering_base`。在P0参数和项目签认未完成时，严格校验必须失败，不允许自动切回`interface_smoke`。

### 4.2 标准交换数据包

每个模块必须从以下工厂创建交换包：

```matlab
packet = common_packet_4_2(moduleId,timeH,cfg,phase);
```

顶层结构固定为：

```text
packet.meta
packet.axis
packet.ports
packet.state
packet.product
packet.service
packet.loss
packet.quality
packet.audit
```

模块不得删除、改名或覆盖这些公共区块。交付前必须执行：

```matlab
validate_module_packet_4_2(packet,moduleId,cfg,true);
```

### 4.3 公共端口字段

所有可控电气端口统一包含：

```text
requestedMW
acceptedMW
actualMW
minMW
maxMW
upCapabilityMW
downCapabilityMW
constraintCode
operatingState
meterPoint
qualityFlag
```

端口功率全部使用非负幅值，方向由端口名称和4.1定义，不通过正负号表达。只有派生量`pBessNetMW=pBessDischargeMW-pBessChargeMW`允许有符号，且不得作为公共总账原始输入。

### 4.4 统一单位和维度

| 对象 | 单位/维度 |
|---|---|
| 联合层有功、无功 | MW、Mvar |
| 电量 | MWh |
| 时间、步长 | h |
| 氢量、氢流率 | kg、kg/h |
| 算力服务 | MWh-CS |
| 单端口逐时数据 | N×1 |
| 多源逐时数据 | N×S |
| 聚合场景 | N×K |
| 多源联合场景 | N×S×K |
| 场景概率 | 1×K |

4.3内部可以继续使用W、var和s，但公共适配器必须转换为MW、Mvar和h。不得把S×N×K数据静默转置为N×S×K。

## 5. 4.3多源能源模块

### 5.1 负责内容

- 漂浮式风电、漂浮式光伏、潮流能设备级可用功率；
- 停机、复归、换向、故障、维护和降额状态；
- 设备电气端至`MP-03-SOURCE-POI`的集电损耗；
- 源侧辅机功率计算并独立输出；
- 点预测、联合场景、P-Q能力、爬坡和上下调边界；
- 多时间尺度互补性诊断。

### 5.2 输入

| 输入 | 来源 |
|---|---|
| `cfg`、`timeH`、`dtH`、场景信息 | 4.1—4.2 |
| 风速、辐照、温度、流速、海况和平台运动 | 公共数据层 |
| 设备参数和功率曲线 | 参数包/OEM/试验数据 |
| `source.requestedMW` | 4.9或接口联调规则器 |
| 上一时段设备状态 | 4.3自身`stateOut` |

### 5.3 标准输出

```text
packet.meta.moduleId = '4.3'
packet.ports.source
packet.loss.pCollectionLossMW
packet.loss.pSourceAuxLoadMW
packet.loss.collectionLossAlreadyDeducted = true
packet.state.sourceOperatingState
packet.state.sourceConstraintCode
packet.state.stateOut
packet.ports.source.availableScenarioMW（可选扩展字段，维度为 N×S×K）
packet.ports.source.forecastScenarioMW（可选扩展字段，维度为 N×S×K）
```

`source`端口测点固定为`MP-03-SOURCE-POI`。`actualMW`进入4.4；`available/forecast/scenario`作为4.3专属数据进入4.9；源侧辅机由4.3计算、4.4记账。场景概率和场景ID使用4.2公共`scenario`轴及其元数据，不得新增`packet.scenario`等第十个顶层区块；模块仅可在既有`packet.ports.source`内增加场景功率字段，且不得覆盖公共端口字段。实际量只允许使用实现路径，不能把场景期望值写入`actualMW`。

### 5.4 必须迁出

- 联合演示中的储能、制氢、算力、外送和全系统调度；
- 公共负荷和公共功率平衡；
- 模块自定义经济日历和容量默认值；
- 任何在MP-03之后再次计算的公共变换损耗。

## 6. 4.4公共母线功率平衡模块

### 6.1 负责内容

4.4是唯一公共母线实际功率总账。它不发布设备BOUNDARY，不接收设备REQUEST，也不修改专业状态，只在LEDGER阶段读取实际端口。

公共总账为：

$$
\begin{aligned}
r_t^P={}&P_t^{src}+P_t^{BESS,dis}+P_t^{other}+P_t^{imp}\\
&-P_t^{BESS,ch}-P_t^{ely}-P_t^{DC}-P_t^{export}-P_t^{marine}\\
&-P_t^{src,aux}-P_t^{common,aux}-P_t^{post}-P_t^{spill}.
\end{aligned}
$$

所有项必须为相应模块的`actualMW`。`pUnservedMW`是可靠性统计量，不得同时作为没有物理来源的发电注入掩盖残差。

### 6.2 输入对应关系

| 4.4总账项 | 来源 |
|---|---|
| `pSourceActualMW`、`pSourceAuxLoadMW` | 4.3 RESPONSE |
| `pBessChargeActualMW`、`pBessDischargeActualMW`、`pElectrolyzerActualMW` | 4.5 RESPONSE |
| `pDCFacilityActualMW` | 4.6 RESPONSE |
| `pExportSendActualMW`、`pGridImportActualMW`、`pMarineActualMW` | 4.7 RESPONSE |
| `pSpillActualMW`、`pUnservedMW` | 4.9显式决策/可行性处理 |
| `pCommonAuxMW`、`pPostPOILossMW` | 4.4唯一计算 |

### 6.3 标准输出

```text
packet.meta.moduleId = '4.4'
packet.meta.phase = 'LEDGER'
packet.state.pBusResidualMW
packet.state.pSpillActualMW
packet.state.pUnservedMW
packet.state.balanceStatus
packet.loss.pCommonAuxMW
packet.loss.pPostPOILossMW
packet.audit.lossLedger
```

### 6.4 必须删除或旁路

- 4.4内部的`SourceUnit/StorageUnit/ComputingUnit/LoadUnit`设备状态；
- 有符号储能单端口；
- PUE、SOC、储氢和设备效率重算；
- 把MWh直接作为MW备用功率；
- 按额定容量估算并重复扣除全部系统损耗；
- 用隐式弃电、备用或缺供自动把残差改为零。

## 7. 4.5电池—制氢—储氢模块

### 7.1 负责内容

- 电池充放电功率、效率、SOC/SOH、爬坡和互斥状态；
- 电解槽最小负荷、启停、爬坡、SEC及设施总功率；
- 产氢量、储氢库存、可交付氢上限和初末状态；
- 构网备用和黑启动安全边界的调度代理接口。

### 7.2 正式端口

| 端口 | 方向 | 测点 |
|---|---|---|
| `bessCharge` | 公共母线消耗 | `MP-04-COMMON-BUS` |
| `bessDischarge` | 公共母线注入 | `MP-04-COMMON-BUS` |
| `electrolyzer` | 制氢设施消耗 | `MP-ELECTROLYZER` |

三个端口均保留`requested/accepted/actual`。不得将充电和放电合成一个有符号公共输入。

### 7.3 状态和产品输出

```text
packet.state.bessSOC
packet.state.bessEnergyMWh
packet.state.h2InventoryKg
packet.state.electrolyzerOn
packet.product.h2ProductionKg
packet.product.h2AvailableForDeliveryKg
```

电解SEC边界固定为制氢设施电气入口总电耗，并明确是否包含BOP、纯化和压缩。V2公共配置采用`FACILITY_TOTAL_INCLUDES_BOP_PURIFICATION_COMPRESSION`；更换设备参数时只能更新参数值和证据，不得在4.4重复增加同一电耗。

### 7.4 氢交付两阶段提交

1. 4.5根据期初库存和本时段产氢发布`h2AvailableForDeliveryKg`；
2. 4.7依据管道/船运通道和REQUEST形成`h2DeliveredKg`；
3. 4.5读取4.7实际提取量，提交期末`h2InventoryKg`；
4. 4.7物流损耗只减少最终交付量，不反向减少4.5产氢量。

### 7.5 必须迁出

- `wind_power_mw`和全系统可用电力；
- 公共刚性负荷、电力外送、系统弃电和全系统平衡；
- 管道/船运最终交付损耗和收益；
- 电价驱动的跨端口最优分配。

## 8. 4.6绿色算力负荷模块

### 8.1 负责内容

- 刚性、可延期、弹性和连续块任务；
- 任务到达、队列、截止时间、SLA和期末状态；
- IT功率、冷却/泵/网络/存储辅助功率和设施总功率；
- 数据中心功率上下界、爬坡及上下调能力；
- 算力服务完成量和约束原因。

### 8.2 正式端口和输出

```text
packet.ports.dcFacility
    meterPoint = MP-DC-FACILITY
packet.state.computeQueueMWhCS
packet.service.computeServedMWhCS
packet.service.pITActualMW
packet.service.pDCAuxActualMW
```

设施端口必须满足：

$$
P_t^{DC}=P_t^{IT}+P_t^{DC,aux}.
$$

4.4只读取`dcFacility.actualMW`，不得再乘PUE。4.7只读取`computeServedMWhCS`并施加光纤/通信交付限制，不得用设施功率反算服务量。

### 8.3 服务单位冻结

- `MWh-CS`：标准化算力服务量，用于任务、交付和收益；
- `MWh-IT`：IT设备耗电量，只用于能耗和效率分析；
- 4.7正式交付字段固定为`computeDeliveredMWhCS`。

### 8.4 必须迁出

- 绿色电与稳定电的系统级分配；
- 电力外送、制氢或其他价值通道收益；
- 联合模式下的独立净收益最大化；
- 与4.1不同的时间步长、初始队列和容量默认值。

## 9. 4.7多形态输出模块

### 9.1 负责内容

- 海缆送端容量、登陆点接纳、送受端损耗和电力结算量；
- 允许时的外购电通道；禁止外购电时端口固定为零；
- 氢管道/船运能力、物流损耗和最终交付量；
- 光纤带宽、延迟、可用率和算力服务交付量；
- 海洋综合用能刚性/柔性需求、实际服务和未供量。

### 9.2 正式电气端口

| 端口 | 方向 | 测点 |
|---|---|---|
| `exportSend` | 公共母线消耗 | `MP-CABLE-SEND` |
| `gridImport` | 公共母线注入 | `MP-CABLE-SEND` |
| `marine` | 公共母线消耗 | `MP-04-COMMON-BUS` |

当`cfg.commonBus.allowGridImport=false`时，`gridImport`的requested、accepted、actual和max必须全部为0。

### 9.3 标准交付输出

```text
packet.service.pCableReceiveMW
packet.service.cableLossMW
packet.product.h2DeliveredKg
packet.service.computeDeliveredMWhCS
packet.service.marineUnservedMW
```

`exportSend.actualMW`进入4.4；`pCableReceiveMW`只用于结算和LCA。`h2DeliveredKg`和`computeDeliveredMWhCS`不进入电功率平衡。

### 9.4 必须删除或旁路

- `compute_load.py`中的PUE和IT功率重算；
- `hydrogen_output.py`中的电解功率、产氢和库存递推；
- `balance.py`中的另一套全系统平衡；
- `objectives.py`中的总目标权衡；
- 4.7 YAML中与`common_case_v2`冲突的联合运行默认值。

上述历史文件可保留作为单元测试原型，但联合入口不得调用其重复功能。

## 10. 损耗与辅机唯一账本

| 项目 | 唯一计算模块 | 进入4.4方式 | 禁止重复位置 |
|---|---|---|---|
| 风光潮集电损耗 | 4.3 | 已在源侧POI前扣除，仅供审计 | 4.4不得再扣 |
| 源侧设备辅机 | 4.3计算、4.4记账 | `pSourceAuxLoadMW` | 不得隐藏进源侧净发电后再扣 |
| 公共变换/母线损耗 | 4.4 | `pPostPOILossMW` | 4.3、4.5—4.7不得重算 |
| 公共岛级辅机 | 4.4 | `pCommonAuxMW` | 不得混入海洋基础负荷 |
| 电池效率损耗 | 4.5 | 进入SOC状态方程 | 4.4不增加附加电池损耗 |
| 制氢设施内部电耗 | 4.5 | 形成`electrolyzer.actualMW` | 4.4不追加SEC/BOP电耗 |
| 数据中心辅助功率 | 4.6 | 已包含于`dcFacility.actualMW` | 4.4和4.7不再乘PUE |
| 海缆损耗 | 4.7 | 送端进4.4，受端用于结算 | 不回减4.4送端功率 |
| 氢物流损耗 | 4.7 | 减少最终交付量 | 不修改4.5产氢量 |

## 11. 状态所有权与提交规则

| 状态 | 唯一所有者 | 其他模块权限 |
|---|---|---|
| 风光潮运行/复归状态 | 4.3 | 只读 |
| 电池SOC/SOH | 4.5 | 只读 |
| 电解槽启停和复归 | 4.5 | 只读 |
| 氢库存 | 4.5 | 4.7只返回实际提取量 |
| 算力任务队列/SLA | 4.6 | 4.7只返回交付限制 |
| 海缆/光纤/物流可用状态 | 4.7 | 只读 |
| 调度计划和REQUEST | 4.9 | 专业模块只裁剪和执行 |

功率表示区间平均值；SOC、库存、队列和设备状态表示区间末状态。不同分辨率之间转换时，功率采用能量守恒平均，累计量求和，状态取期末值并保留事件标志；故障、停机和复归状态禁止线性插值。

## 13. 联合验收清单

### 13.1 公共标准

- [ ] `cfg.meta.schemaId=BLUE_HUB_CH4_SCHEMA_V2`；
- [ ] `cfg.meta.schemaVersion=2.0.0`；
- [ ] `validate_common_schema_4_2(cfg,true)`通过；
- [ ] 所有数据包的`caseId/caseMode/parameterVersion`一致；
- [ ] 联合运行未使用模块静默默认值；
- [ ] 所有假设参数保留规定标注。

### 13.2 时间、场景和维度

- [ ] 所有模块共享相同`timeH`和`dtH`；
- [ ] 场景ID和概率一一对应，概率和为1；
- [ ] 多源联合场景维度为N×S×K；
- [ ] 5 min、15 min和1 h聚合满足能量守恒；
- [ ] 状态和事件没有线性插值；
- [ ] 场景、模块和时段之间无数据错位。

### 13.3 端口与测点

- [ ] 所有可控端口具备requested/accepted/actual；
- [ ] RESPONSE阶段三个值均有限且非负；
- [ ] actual位于端口上下界内；
- [ ] 4.3源侧测点为MP-03；
- [ ] 电池和海洋用能使用MP-04；
- [ ] 电解槽、数据中心和海缆使用各自冻结测点；
- [ ] 4.4只读取actual，不读取forecast、available或requested代替实际值。

### 13.4 损耗与状态

- [ ] 集电损耗、源侧辅机、公共损耗、电池损耗、数据中心辅机和海缆损耗均只计算一次；
- [ ] SOC满足功率、效率、能量和初末状态约束；
- [ ] 氢库存只由4.5递推，且使用4.7实际提氢量；
- [ ] 算力队列只由4.6递推；
- [ ] PUE只在4.6计算；
- [ ] 4.7不重算产氢和IT功率。

### 13.5 功率平衡

- [ ] 4.4逐时残差小于`cfg.commonBus.balanceToleranceMW`；
- [ ] 4.4未通过隐式弃电、缺供或储能响应掩盖残差；
- [ ] 送端功率进入4.4，受端功率不回接公共母线；
- [ ] 氢和算力服务交付量不作为电气端口；
- [ ] 外购电禁用时所有gridImport字段为0；
- [ ] 不可行方案明确返回INFEASIBLE或显式松弛，不静默继续。

### 13.6 方案比较

- [ ] 单一外送与协同方案使用同一场址、资源和源侧装机；
- [ ] 两方案共享设备可用率、故障序列和随机种子；
- [ ] 两方案共享币值年、评价期和LCA边界；
- [ ] 交付点收益与母线功率口径没有混用；
- [ ] 所有差异来自调度、容量配置或价值通道，而不是输入口径变化。

## 15. 完成判据

只有同时满足以下条件，才可判定4.3—4.7完成对接：

1. 每个模块职责与本清单一致，不存在第二个功率平衡、第二个PUE、第二个储氢库存或第二个跨端口目标函数；
2. 所有联合输入来自`common_case_v2`，所有交换包通过V2校验器；
3. 从BOUNDARY到STATE_COMMIT的多时段闭环能够连续运行；
4. 4.4逐时闭合且损耗只计算一次；
5. 任一版本、单位、维度、测点、负端口或缺失实际值错误都会被阻止；
6. 单一外送和源储算用协同方案可以在同一公共输入下公平比较。
