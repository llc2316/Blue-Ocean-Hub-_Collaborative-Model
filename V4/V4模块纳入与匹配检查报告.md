## 5. 模块匹配关系

| 模块 | 与4.1—4.2的关系 | V4处理 |
|---|---|---|
| 4.3 | 原模型使用W、var、s及内部测点 | 只做MW/Mvar/h、时间步长和`MP-03`端口映射，不改变设备方程 |
| 4.4 | 唯一公共母线总账 | 使用冻结的actual-only平衡，避免隐式备用和重复损耗 |
| 4.5 | 电池、制氢、储氢状态模型 | 通过响应适配器提交功率与状态，跨模块决策留给4.9 |
| 4.6 | DC-only算力模型 | 先发布边界，接收4.9逐时请求后调用原求解器；PUE只在4.6计算 |
| 4.7 | 电力、氢、算力、海洋用能输出 | 区分海洋完整需求和可分配功率，返回实际服务与未供量 |
| 4.8 | 评价责任所有者 | 只读取状态提交后的实际量，输出经济、环保、可靠性原始目标及KPI |
| 4.9 | 跨模块调度责任所有者 | 生成4.5/4.6/4.7请求，响应后确认实际弃电和分类缺供；当前不声明最优性 |
