# 公共基线模型V2参数可识别性清单

> 结论：大多数运行参数有明确取数路径，但不是取得任意SCADA数据后都能唯一辨识。功率曲线、损耗、辅机和状态阈值需要同步环境量、设备状态、设备端功率和公共汇集点功率；部分参数必须依赖OEM试验或专项传感器。

## 1. 漂浮式风电

| 参数 | 计划数据 | 识别方法 | 判断 |
|---|---|---|---|
| `ratedPower`、`ratedApparentPower`、`hubHeight` | OEM铭牌、型式认证、设计文件 | 直接冻结 | 可取得 |
| `powerCurveWind/powerCurveP` | 轮毂风速、机端有功、状态码、限发指令 | 剔除故障/限发后分箱或OEM曲线 | 有条件可识别 |
| `cutOutWind/restartWind/restartDelay` | 高频风速、保护事件、启停日志 | 事件对齐与状态转移统计 | 有条件可识别 |
| `maxOperatingWave/restartWave` | 波高、平台报警、停复机日志 | 海况—事件联合分析 | 有条件可识别 |
| `maxOperatingPitch/restartPitch` | IMU姿态、保护定值、停复机日志 | OEM定值优先，事件校核 | 需OEM/专项数据 |
| 平台相对风修正 | 风速、纵荡速度、纵摇角速度、轮毂高度 | 同步时序回归或高保真模型降阶 | 有条件可识别 |
| `rampUp` | EMS指令、接受值、实发功率 | 正常运行指令阶跃辨识 | 有条件可识别 |
| `auxiliaryPower` | 独立辅机电表、运行状态 | 分状态统计 | 需独立计量 |
| `arrayLossFraction`或损耗曲线 | 机端总功率、汇集点功率、拓扑和电气量 | 同步差值及潮流校核 | 需MP同步计量 |
| P–Q能力边界 | OEM能力图、并网试验、电压温度 | 厂家曲线/试验冻结 | SCADA不能充分识别 |

## 2. 漂浮式光伏

| 参数 | 计划数据 | 识别方法 | 判断 |
|---|---|---|---|
| `pdcRated/pacRated/apparentPowerRated` | 组件和逆变器铭牌 | 直接冻结 | 可取得 |
| `gstc/tstc/gammaP/bifaciality` | 组件数据表、IEC试验 | 直接冻结 | 可取得 |
| `U0/U1` | POA辐照、组件温度、环境温度、风速 | Faiman模型回归 | 有条件可识别 |
| `iamB0`或IAM曲线 | 组件试验、入射角和辐照 | OEM/IEC数据优先 | 现场难以独立识别 |
| `invEfficiency` | DC/AC同步功率、温度、电压 | 按负载率分箱 | 有条件可识别 |
| 动态姿态辐照修正 | 太阳位置、DNI/DHI/GHI、平台姿态、POA | 几何模型与实测校准 | 需姿态和POA传感器 |
| 污损、退化、失配 | 清洗记录、I–V曲线、组串功率、辐照 | 干预对比和长期退化分析 | 数据不足时不可分辨 |
| 环境停机/复归阈值 | 风浪、平台报警、逆变器日志 | 事件统计/OEM定值 | 有条件可识别 |
| `auxiliaryPower` | 独立站用电表、运行状态 | 分状态统计 | 需独立计量 |
| `cableLossFraction`或损耗曲线 | 逆变器出口和汇集点同步功率 | 差值与电气模型 | 需MP同步计量 |

## 3. 潮流能

| 参数 | 计划数据 | 识别方法 | 判断 |
|---|---|---|---|
| `ratedPower/ratedApparentPower` | OEM铭牌、示范项目资料 | 直接冻结 | 可取得 |
| 涨/落潮功率曲线 | ADCP扫掠面流速、机端功率、方向和状态 | 分方向功率性能测试 | 需项目原始数据 |
| `cutInSpeed/directionDeadband` | 有符号流速、转子/变流器状态 | 静水和启动事件统计 | 有条件可识别 |
| `reorientationDelay` | 流向、偏航/换向状态、功率 | 换向事件对齐 | 有条件可识别 |
| 最大运行流速及复归 | 流速、保护和复归日志 | 事件统计/OEM定值 | 有条件可识别 |
| 波高停机及复归 | 波高和状态日志 | 事件统计/OEM定值 | 有条件可识别 |
| `wakeVelocityFactor` | 上下游ADCP、机组位置、运行组合 | CFD/阵列试验/实测反演 | 普通SCADA不可唯一识别 |
| `biofoulingFactor` | 水下检查、清洗时间、功率曲线漂移 | 清洗前后对比 | 需维护记录 |
| 运行/待机辅机功率 | 独立辅机计量和状态 | 分状态统计 | 需独立计量 |
| 集电损耗 | 机端与汇集点同步功率、电气量 | 差值和网络校核 | 需MP同步计量 |

## 4. 公共接口与联合模型

| 参数/信号 | 必要数据 | 判断 |
|---|---|---|
| `requested/accepted/actual` | EMS原始指令、设备接受值、POI实测 | 三者必须分别留存 |
| `pAvailable` | 环境输入、设备状态、功率曲线 | 不能直接用实发替代 |
| `pForecastAvailable` | 气象海洋预测及模型版本 | 必须做滚动回测 |
| 联合场景与概率 | 同场风、光、流历史及预测误差 | 必须保持源间和时间相关性 |
| `pAuxLoad` | 各源独立辅机计量 | 否则无法与发电损耗分离 |
| 可用/实际集电损耗 | 设备端与公共汇集点同步计量 | 必须统一测点和时钟 |
| P–Q边界 | OEM曲线、并网试验、网络潮流 | 简单加总仅为调度近似 |

## 5. 最小可辨识数据集

每类设备至少同步取得：

1. 环境资源量及传感器质量标志；
2. 设备运行、故障、保护、检修和复归状态；
3. EMS原始请求、设备接受值和实发功率；
4. 设备端、集电线路和公共汇集点同步功率；
5. 独立辅机功率；
6. 设备和控制软件版本；
7. 清洗、维护、故障和参数变更记录。

缺少状态标签或同步测点时，功率曲线、损耗、辅机、可用率和降额通常不能同时唯一辨识，应明确标注 **[假设值，待企业调研校准]**。
