> 本次角色：系统架构工程师（兼优化建模师、仿真分析员）

## 0. 修订结论

源侧模型应定位为“可用性与不确定性边界生成器”，而不是第二套储能控制器或智能调度器。第4章应按下列数据链串联：

```text
4.2 统一时间轴、集合、单位、符号和测点
  ↓
4.3 单源设备模型 → 公共汇集点适配 → 多源聚合、预测场景、互补性与能力边界
  ↓
4.4 显式端口功率守恒 → 形成储能/备用/弃能所需的系统残差
  ↓
4.5 构网型混合储能返回可行P/Q、SOC/SOH、模式和稳定支撑能力
  ↓
4.6—4.7 算力、制氢、外送和海洋用能返回负荷及柔性边界
  ↓
4.8—4.9 EMS进行日前—日内—实时滚动优化，向各子系统发送设定值和备用要求
```

“能源岛功率平衡模型”仅指4.4的守恒接口，不能指代4.3的多能源互补体系、容量/拓扑层的新能源集群设计，也不能指代4.8—4.9的智能调度。

## 1. 现有模型必须纠正的问题

1. **不能用 `1-FOR` 逐时乘功率。** FOR是统计可靠性指标；时序仿真应使用运行/故障状态及修复过程，容量规划可用FOR构造状态转移概率。FOR、MTBF、MTTR未取得厂家或项目数据时标注 **[假设值，待企业调研校准]**。
2. **不能用一个全局海况条件同时清零风、光、潮。** 三类设备的停机、降额和复归阈值不同，应分别由设备状态机处理；阈值来自厂家、型式试验或标准设计工况。
3. **“空间平滑”不是功率折减系数。** 聚合功率必须逐时相加；平滑效应应由相关性、爬坡、变异系数和低出力持续时间等统计指标评价。尾流、遮挡、集电损耗必须进入对应设备/网络模型，不能再乘一个统一 `gamma_smooth`。
4. **功率必须逐时守恒，不能对整段时间先求和。** 原脚本 `sum(P_actual)` 得到的是样本和，既不是逐时功率也不是严格电量；电量应按时间积分。
5. **不得把预测误差加到“实际功率”后称为预测。** 点预测、概率场景、可用功率、EMS设定值和实测功率应分别存储。
6. **源侧不得替EMS决定风/光/潮功率比例。** “风电85%+”是容量配置情景，不是逐时分配规则；示例光伏、潮流占比均为 **[假设值，待企业调研校准]**。
7. **辅机和损耗不能重复计量。** 推荐把发电注入限制为非负，把逐源辅机功耗作为独立消耗端口；所有P/Q必须折算到同一公共汇集点。

### 1.1 MATLAB落实状态（2026-07-16）

上述七项已落实到当前V3接口和三个单源V2调度代理：

1. 三个单源模型优先使用二值`availabilityState`；旧`availability`字段仅兼容二值输入，禁止连续`1-FOR`折减；
2. 风电、光伏和潮流能分别维护独立停机—复归状态机；风电新增波高和平台纵摇角安全边界；
3. 4.3只逐时聚合，互补性由相关性、CV、爬坡和低出力指标诊断；
4. 4.4逐时守恒，电量统一采用时间积分；
5. 原始EMS请求由适配器外部输入，设备接受值、实际值、预测和场景分开存储；
6. 85%+只保留为装机情景，逐时请求由EMS输入；
7. 三个单源模型统一输出发电毛功率、可用/实际集电损耗、POI非负发电注入和独立辅机负荷。

统一设备输出字段为：

## 2. 章节边界与系统架构

### 2.1 4.2：全模型公共语言

- 电源集合：\($i\in\mathcal S=\{w,pv,tc\}$\)；
- 调度时段：\($t\in\mathcal T$\)；联合场景：\($k\in\mathcal K$\)；
- 统一单位：有功W、无功var、能量Wh或J、时间s；
- 统一符号：发电、其他电源和储能放电为母线正注入；用电、外送和储能充电为正消耗；
- 统一测点：`source_collection_bus`，即三类电源汇入能源岛公共母线前的源侧公共汇集点。

调度层建议采用5—15 min时间步，日前层采用15—60 min时间步，电磁暂态/构网层采用独立的毫秒级或更细步长；具体数值均为 **[假设值，待企业调研校准]**。

### 2.2 4.3：源侧负责的内容

- 输出逐源可用功率、接受设定值、实际功率、限发量和不可用损失；
- 输出点预测、预测分位数或保持时间/源间相关性的联合场景；
- 输出P-Q能力、爬坡能力、可上/下调裕度、运行状态、约束原因和数据质量；
- 计算多时间尺度互补性指标；
- 接受EMS的 `P_set`、`Q_set`、备用、限发和运行模式要求。

### 2.3 4.3不负责的内容

- 电池SOC/SOH、超级电容状态、储氢库存和寿命成本；
- 下垂、VSG、虚拟惯量、黑启动、故障穿越和保护协调；
- 电力、氢、算力和海洋用能之间的经济分配；
- 全局容量配置、集电拓扑寻优和海缆/制氢路线决策。

### 2.4 4.4：只负责公共母线守恒

4.4把4.3、4.5、4.6和4.7接到同一功率方程，但不决定谁优先发电、谁优先用电，也不实现构网控制。

## 3. 统一数据契约

每个单源子模型应输出以下字段；设备数据缺失时允许使用公开基准模型，但必须标记 `parameterSetId` 和来源类型。

| 字段 | 含义 |
|---|---|
| `time, sourceId, sourceType` | 时间轴与设备身份 |
| `meterPoint` | 必须为统一公共汇集点或明确可换算测点 |
| `pAvailableGross` | 资源与设备状态允许的毛可用发电功率 |
| `pAuxLoad` | 逐源运行/待机辅机消耗，非负 |
| `pCollectionLoss` | 汇集点之前的集电/变流损耗，非负 |
| `pAvailableAtPOI` | 汇集点可用净注入，非负 |
| `pRequested, pAccepted` | EMS请求及设备在约束下接受的设定值 |
| `pActualAtPOI` | 汇集点实际净发电注入，非负 |
| `pCurtailmentCommand` | 可用功率与接受设定值之差 |
| `pUnavailableLoss` | 故障、维护或环境停机造成的损失 |
| `qMinAtPOI, qMaxAtPOI` | 当前运行点的无功能力边界 |
| `operatingState, constraintCode` | 运行状态与约束原因 |
| `qualityFlag, parameterSetId` | 数据质量与参数版本 |
| `stateOut` | 状态机、复归计时、上一时刻功率等滚动状态 |

三个V2设备模型目前还存在测点和辅机口径不一致，V3适配层应统一为“非负发电注入 + 独立辅机消耗”。若保留有符号净注入方案，则聚合器和功率平衡必须全链路允许负值，不能一处允许、一处截断。

## 4. 单源物理模型与工程化要求

### 4.1 漂浮式风电

调度模型优先采用具体机型的实测/厂家电功率曲线：

\[
$P_{w,t}^{av}=f_{P-v}\!\left(v_{eff,t},s_{w,t}\right)$
\]

其中 \(s_{w,t}\) 是正常、降额、切出、复归、故障或维护状态。平台运动可通过高保真模型得到的等效风速或降额代理进入调度层：

\[
$v_{eff}=v_{hub}-\dot x_{surge}-H_{hub}\dot\theta_{pitch}$
\]

该式仅是小角度、沿轴向的调度级近似；载荷、控制器和六自由度耦合应由OpenFAST/厂家模型验证，不能用它替代认证分析。IEC 61400-3-2:2025规定浮式风机外部条件与工程完整性要求；GB/Z 44047-2024是国内现行设计依据；IEC 61400-12-1:2022用于功率曲线与AEP测试。来源类型：**国际标准/国家标准**。

工程改进：

- 将功率曲线、切入/额定/切出、复归延时、爬坡和P-Q能力参数化；
- 资源骤降时实际功率必须立即受 \(P^{av}\) 约束，不能用普通下降爬坡限制制造“无来源电能”；若研究惯量响应，必须显式建模转子动能；
- 批处理与滚动调用必须通过 `stateIn/stateOut` 保留启停模式、复归计时和上一时刻功率；
- 国产OEM完整动态参数不可得时，可用IEA 15 MW + VolturnUS-S作为公开基准，但标注为 **[公开基准模型，非国产设备参数]**。

### 4.2 漂浮式光伏

推荐采用IEC 61853的 \(P_{mp}(G,T)\) 矩阵或经试验校准的单二极管模型，而不是固定效率：

\[
$P_{pv,t}^{dc}=P_{mp}\!\left(G_{POA,t},T_{m,t}\right)$
\]

再通过逆变器效率曲线、失配、污损、盐雾/退化状态、辅机和集电损耗得到汇集点功率。平台姿态改变入射角和面内辐照；高频波致姿态应在秒级验证模型计算后，以平均损失、分位爬坡或响应面传递给调度层，避免在5 min模型中直接采样秒级姿态造成混叠。

IEA-PVPS Task 13（2025）指出FPV收益不能统一假定，热行为、波浪诱发损失、污损、退化、可用率与运维均需场址数据校准；IEC 61724-1:2021可用于监测输入与性能校准。IEC 61701盐雾试验和IEC 61215设计鉴定不能直接给出项目年衰减率。来源类型：**国际合作组织技术报告/国际标准**。

### 4.3 潮流能

设备级模型优先使用涨潮/落潮分别测试的电功率曲线：

\[
$P_{tc,t}^{av}=f_{flood/ebb}\!\left(|u_{rel,t}|,s_{tc,t}\right)$
\]

机械机理研究可使用：

\[
$P_m=\tfrac12\rho A C_p(\lambda,\beta)|u|^3,\qquad \lambda=\frac{\omega R}{|u|}$
\]

但进入调度模型时，不能在全流速区无限使用立方律，也不能在已使用机端电功率曲线后再次乘 \(C_p\) 和电机/变流器效率。IEC TS 62600-200规定TEC功率性能评估和功率曲线；IEC TS 62600-201:2025规定潮流资源评估；GB/T 41342-2022规定国内现场功率特性测试。来源类型：**IEC技术规范/国家标准/核心期刊论文**。

工程改进：分别建模涨落潮曲线、方向死区/自动调向时间、尾流速度折减、生物附着、运行/待机辅机、环境停机与复归状态。

## 5. 4.3多源聚合模型

### 5.1 指令接受与实际功率

设备模型而非聚合器执行约束裁剪：

\[
$0\le P_{i,t}^{acc}\le P_{i,t}^{av},\qquad P_{i,t}^{act}=P_{i,t}^{acc}+e_{i,t}^{trk}$
\]

\[
$P_{i,t}^{curt}=P_{i,t}^{av}-P_{i,t}^{acc}$
\]

聚合器只在统一测点求和：

\[
$P_{src,t}^{x}=\sum_{i\in\mathcal S}P_{i,t}^{x},\quad x\in\{av,acc,act,fcst\}$
\]

上述为项目变量定义与守恒关系。设备约束的具体公式、曲线和参数必须来自厂家、试验或可核验文献。

### 5.2 联合预测与场景

\[
$\mathbf P_{t,k}^{scen}=[P_{w,t,k},P_{pv,t,k},P_{tc,t,k}]^{\mathsf T},\qquad P_{src,t,k}^{scen}=\sum_iP_{i,t,k}^{scen}$
\]

场景必须保留共同天气、昼夜周期、潮汐相位、时间持续性以及源间相关性，不能独立抽样三个边缘分布再拼接。可采用经滚动回测验证的Copula、分位数回归或场景生成方法；算法及误差参数为 **[需查证文献支撑/待场址数据校准]**。

### 5.3 互补性评价

互补性不是固定乘数。至少输出：

- 多时间窗口下源间功率/爬坡相关性；
- 聚合功率变异系数 \(CV=$\sigma(P_{src})/\mu(P_{src})$\)；
- 1%、5%、95%、99%爬坡分位数；
- 联合低出力频率、持续时间和能量缺口；
- 外送通道利用率、弃能量和储能残差分布；
- 数据覆盖率及场景校准度。

经验P05只能称为“低分位出力”，不能直接称为容量信用或可靠容量；ELCC需在完整负荷与可靠性模型中计算。互补指标应按连续有效数据段计算，不能跨数据空洞求差分。Jurasz等的综述指出互补性需同时考虑时间、空间和评价指标；单一相关系数不足以证明工程价值。来源类型：**核心期刊综述**。

### 5.4 P-Q与快速支撑能力接口

4.3只汇总厂家/设备模型在公共汇集点给出的能力边界：

\[
$Q_{src,t}^{min}=\sum_iQ_{i,t}^{min},\qquad Q_{src,t}^{max}=\sum_iQ_{i,t}^{max}$
\]

仅当网络电压、电流、变压器和海缆约束未被触发时才可直接加总。源侧还应向4.5/EMS报告 `rampUp/rampDown`、有功备用、穿越可用状态和控制模式；不得据统一视在功率圆虚构厂家P-Q能力。

## 6. 4.4能源岛多端口功率平衡

公共母线有功守恒为：

\[
$P_{src,t}^{act}+P_{other,t}+P_{st,t}+P_{grid\_import,t}=P_{export,t}+P_{el,t}+P_{comp,t}+P_{marine,t}+P_{aux,t}+P_{postPOI\_loss,t}+P_{spill,t}+\varepsilon_t^{bal}$
\]

其中 \(P_{aux}\) 包括源侧适配器上报的辅机负荷和能源岛公共辅机；`postPOI_loss`只包含公共汇集点之后的损耗，避免与单源集电损耗重复。

在4.5尚未返回实际储能功率时：

\[
$P_{st,t}^{req}=P_{demand,t}-P_{src,t}^{act}-P_{other,t}-P_{grid\_import,t}$
\]

正值表示需要储能放电，负值表示存在充电或弃能机会。它是系统残差，不是储能必然可执行的指令；4.5受功率、能量、SOC、温度、寿命和构网约束。若储能饱和，4.4显式接收EMS/可靠性模块给出的 `pUnservedLoad`、`pSpill` 或允许的 `pGridImport`，但4.4本身不分配这些量。

场景下应命名为“净负荷残差场景”而非“储能指令”：

\[
$P_{res,t,k}=P_{demand,t}-P_{src,t,k}^{scen}-P_{other,t}$
\]

风险水平、备用和储能计划由4.8—4.9决定。

## 7. 与4.5构网型混合储能的接口

### 7.1 4.3/4.4发送

- 实时与预测残差、联合净负荷残差场景；
- 源侧P-Q边界、爬坡和备用裕度；
- 运行状态、约束原因、质量标志和保护状态；
- 快慢残差诊断（仅用于选型/容量研究，不作为控制指令）。

### 7.2 4.5返回

- `P_st_actual, Q_st_actual`；
- 可充/可放功率与可用能量边界；
- SOC、SOH、温度、寿命与储氢状态；
- 构网/跟网/停机模式，黑启动、备用和故障穿越可用性。

NREL构网型逆变器路线图区分依赖外部电压源的GFL与能够建立电压/频率的GFM，并将频率、电压、保护、穿越和模型验证列为关键问题。因此，毫秒—秒级电压频率稳定属于4.5及Simulink快速动态层，4.3不得重复实现。来源类型：**国家实验室技术报告**。

## 8. 与4.8—4.9智能调度的闭环

每个滚动周期：

1. 4.3上报可用功率、预测/场景、P-Q/爬坡/备用边界和状态；
2. 4.5—4.7上报储能、算力、制氢、外送和海洋用能可行域；
3. 4.8—4.9优化得到逐源P/Q设定值、储能计划、端口负荷和备用；
4. 各设备状态机返回接受设定值、实际功率和约束原因；
5. 4.3聚合实际源侧注入，4.4计算实际残差；
6. 4.5快速控制维持电压频率并返回实际P/Q；
7. 4.4计算闭环不平衡，EMS在下一周期滚动修正。

建议形成“日前场景优化—日内滚动修正—秒级/毫秒级本地构网控制”的三层结构。具体调度周期为 **[假设值，待企业调研校准]**。

## 9. 项目实例对模型的启示

### 9.1 山东半岛南3号深远海漂浮式光伏实证

国家发展改革委转引资料显示，该500 kW项目与300 MW海上风电同场，采用环形浮体、系泊、定制组件并共用风光送出系统。它证明模型必须显式描述空间安全边界、系泊/波浪状态、共享送出与项目级测点；报道中的冷却增益只适用于该项目，不能作为通用效率增益。来源类型：**政府官网**。

### 9.2 “海油观澜号”深远海漂浮式风电

公开企业材料可用于构造深远海平台、远距离送出、辅助负荷和运维场景，但未公开的完整功率曲线、控制器和P-Q参数不得补造，应标注 **[需企业调研/厂家资料]**。来源类型：**上市公司报告**。

### 9.3 舟山LHD潮流能项目

舟山市政府资料显示LHD项目实现兆瓦级长期并网；后续政府机构资料披露全功率变流接入和预测曲线进入日前计划。它直接支持“潮流曲线—预测—EMS设定值—变流器—公共母线”接口链，但项目功率曲线、可用率、辅机和维护参数仍需原始试验/SCADA校准。来源类型：**政府官网**。

## 10. MATLAB/Simulink V3迭代

### 10.1 必须迭代

现有 `multi_source_complementarity_v2.m` 虽已明确章节边界，但仍同时承担4.3聚合、4.4平衡和互补指标，且三个单源输出契约不一致。V3拆分为：

```text
设备子模型/适配器
  → ../互补体系模型/source_aggregation_4_3_v3.m（4.3）
  → ../互补体系模型/source_complementarity_metrics_v3.m（4.3指标）
  → bus_balance_4_4_v3.m（4.4）
  → marine_source_hub_v3.m（跨模块编排入口）
  → 外部4.5储能模型与4.8—4.9 EMS
```

### 10.2 时间尺度

- **资源/设备验证层**：秒级或更细；浮式风电可对接OpenFAST，光伏姿态/温度和潮流换向状态在此验证；
- **调度代理层**：5—15 min，采用能量守恒平均，并保留区间最小/最大、爬坡、停机占比和质量标志；
- **EMS层**：日前/日内滚动优化；
- **快速动态层**：Simulink中实现4.5构网、P/Q、电压、频率、故障和保护。

不得把状态量和启停信号简单线性插值；降采样需同时输出能量平均、极值和状态占比。

### 10.3 状态与测试

所有设备函数增加 `stateIn/stateOut`，保证分段滚动运行与一次批处理一致。集成测试至少检查：

- 单位、维度、时间轴和测点一致；
- 发电注入非负，辅机进入独立负荷端口；
- 公共汇集点前后损耗不重复；
- 实际功率不超过可用功率；
- 资源骤降不产生虚假能量；
- 数据缺口不产生跨段假爬坡；
- 4.4逐时守恒；
- 4.5饱和后显式形成失负荷或弃能，而不是隐藏在误差中；
- 分段调用与batch调用状态一致。

当前工作区未检测到MATLAB/Octave运行时，V3只能完成代码静态检查；动态仿真、Simulink构网控制和数值结论仍需在具备相应工具箱的环境验证。
