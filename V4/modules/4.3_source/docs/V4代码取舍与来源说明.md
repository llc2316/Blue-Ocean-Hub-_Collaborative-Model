# 4.3 V4代码取舍与来源说明

## 整理结论

V4不再区分`runtime`和`validation`。两套目录中承担相同职责的代码已经合并为一套正式模型，避免后续修改时出现同名函数版本漂移。

| V4职责 | 采用来源 | 取舍 |
|---|---|---|
| 风电、光伏、潮流能设备模型 | 公共基线V2 | 与Version3镜像一致，只保留一份 |
| 设备输出适配、源侧聚合 | 公共基线V2 | 与Version3镜像一致，只保留一份 |
| 互补性指标 | Version3 | 采用字段更完整的增强实现 |
| 组合比较、场景覆盖、状态字典、联合场景校验 | Version3 | 属于验收与分析所需的新增能力 |
| 4.3公共基线演示及其平衡辅助函数 | 公共基线V2 | V4适配器需要它们生成源侧数据；辅助函数不替代正式4.4模块 |
| Version3测试与演示 | Version3 | 用于4.3独立验收 |
| 正式模型文档 | 两套资料择优保留 | 保留完整报告、验收说明、参数清单和模型说明 |

## 已移除内容

- runtime/validation中的同名重复模型；
- `4.3初稿_V3.md`；
- `generate_report_artifacts_4_3_v3.m`；
- 两套旧README和旧目录本身。

原始开发文件仍保留在4.3原模块目录中，本次删除仅发生于`V4整合/modules/4.3_source`候选交付目录。
