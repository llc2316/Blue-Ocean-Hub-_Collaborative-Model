# 4.3 多能源互补体系：多源整合输出与跨模型耦合（V2.3）

> 本次角色：系统架构工程师（兼优化建模师）  
> 适用对象：漂浮式风电（主体）+ 漂浮式光伏（互补）+ 潮流能（预留）  
> 章节定位：承接4.2，向4.4、4.5及4.8—4.9提供源侧统一边界  
> 版本日期：2026-07-15

## 4.3.1 模型定位与边界

多能源互补体系是源侧的“统一适配、聚合和边界生成模块”。它把风、光、潮三个设备级子模型转换为同一时间轴、同一单位和同一公共汇集点下的可比较量，形成可供能源岛功率平衡、构网型混合储能和智能调度共同调用的标准接口。

本节回答四个问题：

1. 各类电源在当前环境和设备状态下能够输出多少；
2. 三类电源聚合后能够向公共母线实际注入多少；
3. 聚合出力存在多大波动、预测误差和备用空间；
4. 下游模型可在什么边界内安全调用源侧资源。

本节不计算公共母线最终功率平衡，不决定储能充放电，不实现构网控制，也不决定电力、制氢、算力和海洋用能之间的价值分配。上述功能分别属于4.4、4.5和4.8—4.9。

```text
4.2 公共参数、时间轴、单位、测点与状态字典
  ↓
单源模型：风电 / 光伏 / 潮流能
  ↓
4.3 测点适配 → 多源聚合 → 联合预测 → 互补诊断 → 源侧可行域
  ├─→ 4.4：实际源侧注入、辅机端口、损耗口径
  ├─→ 4.5：源侧P/Q边界、波动与备用需求信息
  └─→ 4.8—4.9：预测场景、逐源可行域、爬坡与调节余量
          ↓
      EMS逐源请求返回4.3执行
```

## 4.3.2 输入、状态与统一测点

令电源集合为

\[
$\mathcal S=\{w,pv,tc\},\quad t\in\mathcal T,\quad k\in\mathcal K$
\]

其中，\(w\)、\(pv\)、\(tc\)分别表示漂浮式风电、漂浮式光伏和潮流能，\(k\)表示联合场景。

各单源模型至少向4.3返回：

| 接口量 | 含义 | 数据来源类型 |
|---|---|---|
| \($P_{i,t}^{av,gross}$\) | 设备端资源可用毛功率 | 单源物理模型、厂家功率曲线、示范项目SCADA |
| \($P_{i,t}^{loss,col}$\) | 公共汇集点之前的集电损耗 | 设计院计算、设备试验、项目实测 |
| \($P_{i,t}^{aux}$\) | 单源辅机功率 | 厂家资料、项目SCADA |
| \($P_{i,t}^{fcst}$\) | 可用功率点预测 | 预测模型、气象海洋预报 |
| \($Q_{i,t}^{min/max}$\) | 当前运行点的无功能力边界 | 厂家P–Q能力图、并网试验 |
| \($z_{i,t}$\) | 正常、降额、停机、复归、换向等状态 | 单源状态机、SCADA |
| \($c_{i,t}$\) | 资源、环境、故障、限发等约束原因 | 单源状态机、EMS日志 |

统一源侧公共汇集点的可用注入定义为：

\[
$P_{i,t}^{av}=\max\left(0,P_{i,t}^{av,gross}-P_{i,t}^{loss,col}\right)$
\]

该式是项目的测点换算定义，不是设备效率公式。单源辅机\(P_{i,t}^{aux}\)不隐藏在发电功率中，而作为独立消耗端口交给4.4，避免在4.3和4.4重复扣减。

集电损耗函数和参数须由设计计算或实测校准；没有可靠来源时标注 **[假设值，待企业调研校准]**。

## 4.3.3 单源请求、接受与实际输出

4.8—4.9向各电源发送原始请求\($P_{i,t}^{req}$\)。4.3调用设备状态机、可用功率和爬坡边界，得到设备可接受指令\($P_{i,t}^{acc}$\)：

\[
$0\le P_{i,t}^{acc}\le P_{i,t}^{av}$
\]

正常受控运行时，可写为：

\[
$-R_i^{dn}\Delta t\le P_{i,t}^{acc}-P_{i,t-1}^{acc}\le R_i^{up}\Delta t$
\]

资源骤降、保护动作和故障停机优先服从实时可用边界，不应被常规爬坡约束阻止。\($R_i^{up/down}$\)须来自厂家控制参数或并网试验；缺失时为 **[假设值，待企业调研校准]**。

实际输出定义为：

\[
$P_{i,t}^{act}=P_{i,t}^{acc}+e_{i,t}^{trk}$
\]

其中\($e_{i,t}^{trk}$\)是跟踪误差。对于不含转子或直流侧显式储能状态的调度代理，应满足：

\[
$0\le P_{i,t}^{act}\le P_{i,t}^{av}$
\]

为保证结果可解释，损失应分项记录：

\[
$P_{i,t}^{curt}=P_{i,t}^{av}-P_{i,t}^{acc}$
\]

调度限发、设备不可用、环境停机、集电损耗和辅机耗电必须分别统计，不能全部归入“弃电”。

## 4.3.4 多源聚合输出

在统一公共汇集点，聚合量为：

\[
$P_{src,t}^{x}=\sum_{i\in\mathcal S}P_{i,t}^{x},\quad x\in\{av,fcst,req,acc,act\}$
\]

4.3对外的核心物理输出为\($P_{src,t}^{act}$\)，它代表源侧向4.4公共母线的实际有功注入。聚合可用功率和实际功率之间的差额为：

\[
$H_{src,t}^{up}=P_{src,t}^{av}-P_{src,t}^{act}$
\]

\($H_{src,t}^{up}$\)仅表示当前资源和设备边界下的理论上调余量。其能否作为可靠备用，还需考虑持续时间、预测误差、通信、爬坡和设备状态，并由4.8—4.9施加备用约束。

源侧无功能力简化汇总为：

\[
$Q_{src,t}^{min}=\sum_iQ_{i,t}^{min},\qquad Q_{src,t}^{max}=\sum_iQ_{i,t}^{max}$
\]

该加总只适用于同一测点且未触发线路、变压器、电压和电流约束的调度近似。工程校核必须使用厂家P–Q能力图和交流潮流模型；不能用统一视在功率圆替代全部设备能力。

## 4.3.5 联合预测与不确定性输出

4.3应接收或生成保持源间相关性和时间持续性的联合场景：

\[
$\mathbf P_{t,k}^{scen}=[P_{w,t,k},P_{pv,t,k},P_{tc,t,k}]^{\mathsf T}$
\]

\[
$P_{src,t,k}^{scen}=\sum_{i\in\mathcal S}P_{i,t,k}^{scen},\qquad \sum_k\pi_k=1,\quad $\pi_k\ge0
\]

三个边缘分布不能独立抽样后直接拼接，否则会破坏共同天气过程、昼夜周期、潮汐相位和连续低出力事件。可选方法包括经滚动回测验证的Copula、历史轨迹重采样或概率预测场景生成。来源类型：**核心期刊论文**。

- Pinson等关于短期风电概率预测到统计场景转换的研究；
- Jurasz等关于可再生能源互补性评价的综述。

具体算法、分布类型、场景数和概率评价指标应根据数据量与调度方法确定；未完成原文和数据验证前标注 **[需查证文献支撑]** 或 **[假设值，待企业调研校准]**。

## 4.3.6 多时间尺度互补性评价

多能源互补不能只用一个相关系数判断。4.3至少输出以下诊断量：

1. 不同时间窗口下的源间相关系数；
2. 单源与聚合功率的变异系数；
3. 聚合功率爬坡率及其高分位；
4. 联合低出力频率、持续时间和能量缺口；
5. 聚合功率经验低分位；
6. 相对“仅风电”基准的波动降低量与可用出力改善量。

聚合爬坡率可定义为：

\[
$r_{src,t}=\frac{P_{src,t}^{act}-P_{src,t-1}^{act}}{\Delta t}$
\]

低出力事件可定义为：

\[
$I_t^{low}=\mathbb I\left(P_{src,t}^{act}<\alpha P_{src}^{rated}\right)$
\]

阈值\($\alpha$\)、统计窗口和分位水平是评价设定，均需敏感性分析，属于 **[假设值，待企业调研校准]**。经验低分位不应直接称为容量信用；容量信用需另建可靠性模型并选择ELCC等方法。

## 4.3.7 与4.4能源岛功率平衡模型的耦合

4.3向4.4提供：

- 聚合实际注入\($P_{src}^{act}$\)；
- 逐源辅机功率\($P_i^{aux}$\)；
- 已在公共汇集点之前扣除的集电损耗及测点标识；
- 源侧数据质量、运行状态和约束原因。

4.4使用这些量与外送、制氢、算力、海洋用能、公共辅机、母线后损耗和其他注入共同闭合功率守恒。4.3不得预先计算“储能应充放多少”，否则会与4.4的显式端口守恒重复。

因此，“能源岛功率平衡模型”只能指4.4的系统耦合骨架；“多能源互补体系”则指4.3的源侧整合与边界生成，两者不能互相替代。

## 4.3.8 与4.5构网型混合储能中枢的耦合

4.3不直接向储能下达充放电指令。标准链路为：4.3提供实际源侧注入和波动边界，4.4形成系统净功率缺口，4.5根据自身可行域响应。

为支持4.5规划和控制，4.3可同步提供：

| 4.3输出 | 4.5用途 |
|---|---|
| 聚合爬坡率及分位数 | 估计短时储能功率需求 |
| 联合低出力持续时间 | 估计长时储能能量需求 |
| 点预测误差与联合场景 | 设置备用和SOC安全裕度 |
| \(Q_{src}^{min/max}\) | 协调源侧与储能无功支撑 |
| 设备状态与质量标志 | 故障、通信异常时降级运行 |

毫秒—秒级的下垂、虚拟同步机、虚拟惯量、电压频率支撑、黑启动和故障穿越均属于4.5及其底层控制，不在4.3实现。4.3中的滤波或频带分解若被使用，只能作为波动需求诊断，不能直接当作储能容量配置或控制器参数。

## 4.3.9 与4.8—4.9智能调度体系的双向耦合

4.3向EMS提供：

\[
$\mathcal B_t^{src}=\{P^{av},P^{fcst},\mathbf P^{scen},P/Q\text{边界},R^{up/down},H^{up},z,c,quality\}$
\]

该集合构成源侧可行域和不确定性边界。EMS结合4.5储能边界和4.6—4.7柔性负荷边界，优化形成\($P_{i,t}^{req}$\)、备用要求和各价值端口计划；4.3再通过各单源状态机返回\($P_{i,t}^{acc}$\)、\($P_{i,t}^{act}$\)和约束原因。

推荐滚动闭环：

```text
① 4.3更新可用功率、预测、联合场景和源侧可行域
② 4.5—4.7上报储能及负荷可行域
③ 4.8—4.9生成逐源请求、储能计划、端口计划和备用要求
④ 4.3执行请求并返回接受值、实际值及受限原因
⑤ 4.4计算实际母线缺口
⑥ 4.5快速响应并返回实际P/Q和状态
⑦ 4.4闭环校验，EMS在下一周期滚动修正
```

“风电85%+”只用于装机容量情景，不能作为每一时刻的固定出力权重。实时出力由资源、设备状态和EMS可行调度共同决定。

## 4.3.10 标准输出结构

建议MATLAB/Simulink接口统一为：

```matlab
sourceOut.meta        % 时间轴、单位、公共测点、参数版本、质量标志
sourceOut.perSource   % available/forecast/requested/accepted/actual
sourceOut.aggregate  % 聚合P/Q、上下调余量、爬坡边界
sourceOut.scenario   % N×S×K联合场景、概率及聚合场景
sourceOut.losses     % 集电损耗、辅机、限发、不可用和环境损失
sourceOut.state      % 运行状态、受限原因、滚动状态
sourceOut.metrics    % 相关性、CV、爬坡、低出力和经验分位
sourceOut.interface  % 面向4.4、4.5和EMS的标准字段
```

4.3函数不应包含`pExport`、`pElectrolyzer`、`pCompute`、`pMarine`、`pStorageActual`、SOC/SOH、价格或收益。这些字段进入4.4—4.9后再参与系统平衡与优化。

## 4.3.11 MATLAB框架迭代建议

现有`multi_source_complementarity_v2.m`同时处理4.3聚合、4.4端口平衡和储能有功接口，适合联调演示，但不宜作为正式章节的单一生产函数。拆分为：

1. `v2/互补体系模型/source_aggregation_4_3_v3.m`：负责统一测点、多源聚合、联合场景和能力边界；
2. `v2/互补体系模型/device_output_to_source_4_3_v3.m`：将三个设备模型转换为统一4.3契约；
3. `v2/互补体系模型/source_complementarity_metrics_v3.m`：负责多时间尺度互补指标；
4. `v2/互补体系模型/demo_source_aggregation_4_3_v3.m`：只验证4.3输入输出契约，不包含储能或负荷端口；
5. `v2/互补体系模型/demo_device_models_to_4_3_v3.m`：验证三个设备模型经适配器进入4.3；
6. `v2/源侧模型/bus_balance_4_4_v3.m`：只负责显式端口守恒和系统残差；
7. `v2/源侧模型/marine_source_hub_v3.m`：作为编排器连接4.3、4.4和外部4.5/EMS；
8. 三个单源V2函数继续负责物理出力与状态机，不在聚合函数中复制设备公式。

该拆分将4.3代码归入“互补体系模型”，将4.4和跨模块编排代码归入“源侧模型”。后续应重点补齐联合场景校验、P/Q网络校核接口、滚动状态持久化和约束原因字典，而不是在4.3中增加储能控制器。

## 4.3.12 验证准则

4.3单元测试至少包括：

1. 三个单源输入的时间轴、单位和测点完全一致；
2. 聚合输出逐点等于各源同口径加总；
3. `requested`、`accepted`、`actual`和`available`次序与异常原因可追溯；
4. 辅机与集电损耗只扣一次；
5. 联合场景保持源间相关性、时间持续性和物理边界；
6. 缺测、停机、换向和极端海况下状态连续且不制造额外能量；
7. 4.3输出可以直接驱动4.4，且不含4.4—4.9专属决策变量；
8. EMS不可行请求被正确裁剪，并返回明确约束原因。

## 4.3.13 本节结论

4.3的最终交付不是一个“把风光潮功率相加”的公式，而是一套可审计的源侧标准接口：单源设备模型决定资源和设备边界，多能源互补模块完成统一测点、聚合、联合不确定性与互补诊断，4.4完成系统功率守恒，4.5完成储能与构网响应，4.8—4.9完成跨端口价值优化。

据此，4.3能够实现多源能源整合输出，又不会与构网型混合储能中枢和智能调度体系发生功能重复。
