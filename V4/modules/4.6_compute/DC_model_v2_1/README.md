# DC_model_v2_1 运行模式

## 联合调度主模型

`run_model.bat` 调用 `model.py`，读取刚性、可延期和可选任务，计算任务服务、刚性未服务、柔性队列/逾期、设施功率、收入和 SLA 惩罚。V4 默认使用本模式。

## 功率跟随对照模型

`run_raw_model.bat` 调用 `raw_model.py`。该模式不读取外生任务到达，假定任务池无限，只根据可用供电、在线基础功率和 IT 容量计算最大可吸纳功率及对应服务量。它不是“全刚性任务模型”，因为没有刚性需求基线，`rigid_unserved_mwh_it` 恒为零。

正确用途：

- 富余绿电最大吸纳能力；
- 设施功率曲线和容量压力测试；
- 与 SLA 主模型比较柔性约束造成的吸纳折减。

如需全刚性任务场景，应使用 `model.py`，将柔性/可选任务设为零，并提供外生刚性到达序列。

两种模型当前都把 `dc_base_power_mw` 作为在线硬约束；正功率请求低于该值会报错。停机、重启和基础设施供电缺口须由 V4 的 4.9 协调层显式处理。
