# 绿色算力负荷模型  

### 前情提要

模型以**算力任务到达、任务队列、截止时间、服务器容量、数据传输、冷却辅助功耗和稳定供电约束**为基础，输出逐时算力服务量、IT 功率、数据中心总功率、任务积压、服务收入及可调节范围。考虑到大模型预训练与普通离线计算任务在可中断性上的差异，暂时将工作负载划分为**刚性任务、可延期可分割任务、可选弹性任务和连续运行块任务四类**。无 DVFS 版本作为主模型，用于描述任务级柔性；DVFS 模块作为可插拔增强层，用于描述处理器频率调整带来的设备级柔性。

---

# 1. 模型边界与总体结构

## 1.1 研究对象

研究对象为海底绿色算力集群。模型描述如下映射链：

\[
$\text{任务到达}\rightarrow\text{数据传输与准备}\rightarrow\text{任务排队和调度}\rightarrow\text{服务器执行}\rightarrow\text{IT功率}\rightarrow\text{冷却及辅助功率}\rightarrow\text{数据中心总功率}\rightarrow\text{算力服务价值}$
\]


## 1.2 时间尺度

调度周期划分为等长时段：

\[
$\mathcal T=\{1,2,\ldots,T\}$
\]

每个时段长度为 \($\Delta t$\)。初始模型建议采用：

\[
$\Delta t=1\ \mathrm{h}$
\]

典型日、典型周和全年模型分别对应 \(T=24\)、\(T=168\) 和 \(T=8760\)。

## 1.3 基本假设

1. 算力任务可按标准计算服务量统一计量，主单位为 MWh-CS，其中 CS 表示标准化 compute service。
2. 刚性任务必须在到达后立即执行；可延期任务允许在截止时间内分时执行；大规模同步训练等任务采用连续块任务建模，不允许被任意切碎。
3. 数据传输先于计算，通过“数据缓存状态”描述，不将当期计算量与当期光纤带宽简单等同。
4. 服务器存在基础功耗，空载时 IT 功率不为零。
5. 冷却和辅助系统存在最低稳定功耗，数据中心总功率不采用全负载范围内固定 PUE 的唯一表达。
6. 绿色电力可能波动甚至归零，因此模型保留稳定供电接口，以保障基础 IT、网络、存储和冷却负荷。
7. 无 DVFS 模型作为主模型；DVFS 仅作为设备级增强模块，不改变任务层、供电层和经济层结构。
8. 采用滚动时域或首尾队列闭合，避免典型日仿真从零积压开始产生冷启动偏差。

# 2. 集合、索引与任务分类

| 符号               | 含义                                            |
| ------------------ | ----------------------------------------------- |
| \($\mathcal T$\)   | 调度时段集合                                    |
| \(t\)              | 时段索引                                        |
| \($\mathcal K^r$\) | 刚性任务类别集合                                |
| \($\mathcal K^f$\) | 可延期、可分割任务类别集合                      |
| \($\mathcal J^b$\) | 连续运行块任务集合                              |
| \(k\)              | 聚合任务类别索引                                |
| \(j\)              | 连续块任务索引                                  |
| \($\mathcal R$\)   | 计算资源类型集合，如 GPU、CPU、显存、内存和网络 |
| \(r\)              | 资源类型索引                                    |
| \($\mathcal L$\)   | DVFS 运行档位集合，仅增强模型使用               |
| \(l\)              | DVFS 档位索引                                   |

聚合任务集合定义为：

\[
$\mathcal K=\mathcal K^r\cup\mathcal K^f$
\]

其中刚性任务最大延期为零：

\[
$L_k=0,\qquad k\in\mathcal K^r$
\]

柔性任务满足：

\[
$L_k>0,\qquad k\in\mathcal K^f$
\]

任务类型建议对应如下：

- 刚性任务：在线推理、实时交互、控制与通信服务；
- 可延期可分割任务：批量推理、数据清洗、离线评测、独立参数搜索、渲染和科学计算子任务；
- 可选弹性任务：无强制合同约束、仅在绿电充足时接入的全国算力任务；
- 连续块任务：大规模同步预训练、长时间多 GPU 训练和不适合频繁抢占的任务。

# 5. 无 DVFS 基准模型

## 5.1 刚性与柔性任务队列

聚合任务队列平衡为：

\[
$q_{k,t}=q_{k,t-1}+A_{k,t}-x_{k,t},\qquad \forall k\in\mathcal K,\ t\in\mathcal T\tag{1}$
\]

初始状态为：

\[
$q_{k,0}=q_k^{init}\tag{2}$
\]

执行量不能超过当前可用任务量：

\[
$0\leq x_{k,t}\leq q_{k,t-1}+A_{k,t}\tag{3}$
\]

任务执行速率约束为：

\[
$0\leq \frac{x_{k,t}}{\Delta t}\leq r_k^{\max}\tag{4}$
\]

## 5.2 最大延期约束

对类别 \(k\) 的累计截止时间约束为：

\[
$\sum_{\tau=1}^{t}x_{k,\tau}+h_{k,t}\geq\sum_{\tau=1}^{t-L_k}A_{k,\tau},\qquad t=L_k+1,\ldots,T\tag{5}$
\]

若不允许逾期，则：

\[
$h_{k,t}=0\tag{6}$
\]

对于刚性任务 \($k\in\mathcal K^r$\)，有 \($L_k=0$\)，因此可直接写成：

\[
$x_{k,t}=A_{k,t},\qquad \forall k\in\mathcal K^r\tag{7}$
\]

柔性任务允许在截止时间之前重新分配，但不能无限延后。

## 5.3 周期闭合与滚动时域

典型日或典型周模型不应默认从零积压开始。采用首尾闭合条件：

\[
$q_{k,T}=q_{k,0}\tag{8}$
\]

若采用滚动时域，则每次优化未来 \(H\) 个时段，只执行首时段决策，并把实际队列状态带入下一轮。式（8）可在滚动优化中删除。

## 5.4 可选弹性任务

可选任务不形成强制队列：

\[
$0\leq x_t^s\leq D_t^s\tag{9}$
\]

此类任务是吸收风电富余功率的主要柔性来源。

# 6. 连续运行块任务模块

大规模同步预训练等任务不能被视为无成本、任意分割的负荷。对于连续块任务 \($j\in\mathcal J^b$\)，定义启动变量 \($z_{j,t}$\)。

强制任务必须启动一次：

\[
$\sum_{t=s_j}^{d_j-H_j+1}z_{j,t}=1,\qquad \alpha_j=1\tag{10}$
\]

可选任务满足：

\[
$\sum_{t=s_j}^{d_j-H_j+1}z_{j,t}=a_j,\qquad \alpha_j=0\tag{11}$
\]

运行状态由启动状态决定：

\[
$y_{j,\tau}=\sum_{t=\max(s_j,\tau-H_j+1)}^{\min(\tau,d_j-H_j+1)}z_{j,t}\tag{12}$
\]

因此任务一旦启动，将连续运行 \($H_j$\) 个时段，不允许模型将其切成多个零散片段。

连续块任务贡献的 IT 功率为：

\[
$P_{t}^{block}=\sum_{j\in\mathcal J^b}p_j^{IT}y_{j,t}\tag{13}$
\]

对应完成收入为：

\[
$R^{block}=\sum_{j\in\mathcal J^b}\pi_j^b a_j\tag{14}$
\]

若后续需要模拟 checkpoint 恢复，可增加中断变量、恢复时间和回滚工作量；本模型第一版通过连续运行约束避免产生“虚假柔性”。

# 7. 数据传输与缓存模型

当期计算使用的数据必须已在此前时段或当前时段传输并进入缓存。

数据缓存状态为：

\[
$H_{k,t}^{data}=H_{k,t-1}^{data}+u_{k,t}^{in}-b_k^{in}x_{k,t},\qquad \forall k,t\tag{15}$
\]

并满足：

\[
$0\leq H_{k,t}^{data}\leq H_k^{data,\max}\tag{16}$
\]

数据可用性约束为：

\[
$b_k^{in}x_{k,t}\leq H_{k,t-1}^{data}+u_{k,t}^{in}\tag{17}$
\]

光纤输入带宽约束为：

\[
$\sum_{k\in\mathcal K}u_{k,t}^{in}\leq B_t^{\max}\Delta t\tag{18}$
\]

输出数据约束为：

\[
$u_{k,t}^{out}\geq b_k^{out}x_{k,t}\tag{19}$
\]

\[
$\sum_{k\in\mathcal K}\left( u_{k,t}^{in}+u_{k,t}^{out}\right)\leq B_t^{\max}\Delta t\tag{20}$
\]

若输入与输出使用不同方向的独立带宽，可分别设置上下行容量。连续块任务的数据加载可通过在 \($s_j$\) 前设置最小缓存量实现。

---

# 8. 多资源容量约束

为避免只用 GPU 功率而忽略 CPU、内存和网络瓶颈，对每类资源 \($r\in\mathcal R$\) 设置：

\[
$\sum_{k\in\mathcal K}\gamma_{k,r}\frac{x_{k,t}}{\Delta t}+\sum_{j\in\mathcal J^b}\Gamma_{j,r}y_{j,t}\leq C_r\tag{21}$
\]

其中：

- \($\gamma_{k,r}$\)：单位类别 \(k\) 服务对资源 \(r\) 的占用；
- \($\Gamma_{j,r}$\)：连续块任务 \(j\) 对资源 \(r\) 的固定占用。

该式能够表达：

- GPU 算力上限；
- CPU 预处理瓶颈；
- HBM 或主内存容量限制；
- 互联和网络容量限制。

若缺少详细参数，第一版可仅保留 GPU/IT 容量约束，其他资源作为后续标定接口。

# 9. IT 功率模型

无 DVFS 时，IT 功率由基础功耗、聚合任务功耗和连续块任务功耗构成：

\[
$P_t^{IT}= P^{base}+\sum_{k\in\mathcal K}\epsilon_k\frac{x_{k,t}}{\Delta t}+\epsilon_s\frac{x_t^s}{\Delta t}+P_t^{block}\tag{22}$
\]

并满足：

\[
$P_t^{IT}\leq A_t^{IT}C^{IT}\tag{23}$
\]

基础功率 \($P^{base}$\) 不应取零，它代表服务器空闲功耗、存储、网络、控制节点以及必须持续运行的基础服务。

---

# 10. 冷却与辅助功率模型

为避免固定 PUE 在低负载区域失真，定义辅助功率 \($P_t^{aux}$\)，并设置：

\[
$P_t^{aux}\geq P^{aux,\min}\tag{24}$
\]

\[
$P_t^{aux}\geq \left(PUE^{\min}-1\right)P_t^{IT}\tag{25}$
\]

数据中心总功率为：

\[
$P_t^{DC}= P_t^{IT}+P_t^{aux}\tag{26}$
\]

由于目标函数会惩罚无意义用电，最优解中有：

\[
$P_t^{aux}=\max\left\{ P^{aux,\min}, \left(PUE^{\min}-1\right)P_t^{IT}\right\}$
\]

该表达可近似反映低负载时泵、换热器、配电和控制系统效率下降的问题。后续若获得海水温度、IT 负载率和泵功率数据，可将式（24）—（25）替换为分段线性冷却曲线。

---

# 11. 供电与电气安全约束

数据中心总功率由绿色供电和稳定供电共同承担：

\[
$P_t^{DC}= P_t^{green}+ P_t^{firm}\tag{27}$
\]

绿色供电上限：

\[
$0\leq P_t^{green}\leq \bar P_t^{green}\tag{28}$
\]

稳定供电上限：

\[
$0\leq P_t^{firm}\leq \bar P_t^{firm}\tag{29}$
\]

电气接口应保留安全裕度：

\[
$P_t^{DC}\leq (1-\sigma)C^{elec}\tag{30}$
\]

若数据中心嵌入能源岛统一优化，\($P_t^{green}$\) 和 \($P_t^{firm}$\) 可由总功率平衡统一决定；本模块只需输出 \($P_t^{DC}$\)。

---

# 12. 功率动态约束

定义 IT 功率变化：

\[
$P_t^{IT}-P_{t-1}^{IT}=g_t^{up}-g_t^{down}\tag{31}$
\]

其中：

\[
$0\leq g_t^{up}\leq R^{up}\Delta t\tag{32}$
\]

\[
$0\leq g_t^{down}\leq R^{down}\Delta t\tag{33}$
\]

该约束用于近似服务器节点唤醒、容器启动、任务重新部署、模型加载和冷却系统调整过程，不允许 IT 功率在相邻时段任意跳变。

---

# 13. 收入与成本模型

## 13.1 算力服务收入

合同任务收入：

\[
$R^{contract}=\sum_{t\in\mathcal T}\sum_{k\in\mathcal K}\pi_{k,t}x_{k,t}\tag{34}$
\]

可选任务收入：

\[
$R^{spot}=\sum_{t\in\mathcal T}\pi_t^s x_t^s\tag{35}$
\]

总算力收入：

\[
$R^{compute}= R^{contract}+R^{spot}+R^{block} \tag{36}$
\]

## 13.2 能源成本与机会成本

\[
$C^{energy}=\sum_{t\in\mathcal T}\left(\lambda_t^{green}P_t^{green}+ \lambda_t^{firm}P_t^{firm}\right)\Delta t\tag{37}$
\]

其中 \($\lambda_t^{green}$\) 可解释为将风电用于算力而不是外送、储能或制氢的机会成本。

## 13.3 变动运维成本

\[
$C^{om}=\sum_{t\in\mathcal T}\left[\sum_{k\in\mathcal K}c_k^{om}x_{k,t}+ c_s^{om}x_t^s\right]\tag{38}$\]

## 13.4 服务质量成本

队列等待成本：

\[
$C^{queue}=\sum_{t\in\mathcal T}\sum_{k\in\mathcal K^f} c_k^q q_{k,t}\Delta t\tag{39}$
\]

逾期成本：

\[
$C^{late}=\sum_{t\in\mathcal T}\sum_{k\in\mathcal K}c_k^{late}h_{k,t}\Delta t\tag{40}$
\]

周期末未完成成本：

\[
$C^{unf}=\sum_{k\in\mathcal K}c_k^{unf}q_{k,T}\tag{41}$
\]

## 13.5 动态调节成本

\[
$C^{dyn}=\sum_{t\in\mathcal T}\left(c^{up}g_t^{up}+c^{down}g_t^{down}\right)+\sum_{j\in\mathcal J^b}\sum_{t\in\mathcal T}c_j^{start}z_{j,t}\tag{42}$
\]

## 13.6 碳排放成本

碳成本仅作用于具有非零排放因子的稳定供电：

\[
$C^{CO_2}= c^{CO_2}\sum_{t\in\mathcal T}EF_t^{firm}P_t^{firm}\Delta t\tag{43}$
\]

纯弃风消纳不额外施加碳惩罚。

# 14. 无 DVFS 模型目标函数

基准模型以最大化算力净运行价值为目标：

\[
$\max Z_0=R^{compute}- C^{energy}- C^{om}- C^{queue}- C^{late}- C^{unf}- C^{dyn}- C^{CO_2}\tag{44}$
\]

约束集合包括式（1）—（33）。

若连续块任务模块启用，则模型包含二进制变量 \($z_{j,t})、(y_{j,t}) 和 (a_j$\)，属于混合整数线性规划（MILP）；若连续块任务被预先固定为刚性负荷，其他约束均为线性连续变量，则模型退化为线性规划（LP）。

# 15. DVFS 增强模块

DVFS 模块描述处理器电压与频率调整带来的设备级柔性。该模块不替代任务延期，而是在任务级柔性的基础上进一步调整功率和吞吐率。

## 15.1 筛选级连续线性 DVFS

定义 DVFS 因子：

\[
$a^{\min}\leq a_t\leq a^{\max}\tag{45}$
\]

额定点为：

\[
$a_0=1$
\]

对于任务类型 \(k\)，定义 DVFS 敏感系数 \($\chi_k\in[0,1]$\)。其最大服务速率为：

\[
$\frac{x_{k,t}}{\Delta t}\leq r_k^{\max}\left[1+\chi_k(a_t-a_0)\right]\tag{46}$
\]

其中：

- 计算密集型任务取较大的 \($\chi_k$\)；
- 内存带宽受限任务取较小的 \($\chi_k$\)；
- 通信受限的大规模同步训练可取 \($\chi_k\approx0$\)。

将基础 IT 功率拆分为不可 DVFS 部分和可 DVFS 部分：

\[
$P^{base}= P^{nonDVFS}+P^{DVFS,0}\tag{47}$
\]

采用额定点附近的线性近似：

\[
$P_t^{DVFS}= P^{DVFS,0}+ \kappa^{P}(a_t-a_0)\tag{48}$
\]

增强模型 IT 功率为：

\[
$P_t^{IT}= P^{nonDVFS}+ P_t^{DVFS}+\sum_{k\in\mathcal K}\epsilon_k\frac{x_{k,t}}{\Delta t}+ \epsilon_s\frac{x_t^s}{\Delta t}+P_t^{block}\tag{49}$
\]

式（46）—（49）在 \($a_t$\) 的局部小范围内保持线性，适用于规划与调度层敏感性分析，不应直接用于预测某型号 GPU 的芯片级节能量。

定义 DVFS 偏离辅助变量：

\[
$u_t^{dvfs}\geq a_t-a_0\tag{50}$
\]

\[
$u_t^{dvfs}\geq a_0-a_t\tag{51}$
\]

DVFS 调整成本为：

\[
$C^{dvfs}= c^{dvfs}\sum_{t\in\mathcal T}u_t^{dvfs}\Delta t\tag{52}$
\]

连续线性 DVFS 增强目标为：

\[
$\max Z_1=Z_0-C^{dvfs}\tag{53}$
\]

## 15.2 分段线性 / 离散 P-state 版本

若获得具体 GPU 的功率—频率—吞吐数据，建议用离散运行档位替代连续线性近似。

定义：

\[
$z_{l,t}^{dvfs}\in\{0,1\}$
\]

\[
$\sum_{l\in\mathcal L}z_{l,t}^{dvfs}=1\tag{54}$
\]

其中每个档位 \(l\) 给定：

- 功率系数 \($\eta_l^{power}$\)；
- 任务类型相关吞吐系数 \($\eta_{k,l}^{service}$\)。

任务服务能力为：

\[
$\frac{x_{k,t}}{\Delta t}\leq r_k^{\max}\sum_{l\in\mathcal L}\eta_{k,l}^{service}z_{l,t}^{dvfs}\tag{55}$
\]

可 DVFS 功率为：

\[
$P_t^{DVFS}= P^{DVFS,0}\sum_{l\in\mathcal L}\eta_l^{power}z_{l,t}^{dvfs}\tag{56}$
\]

该形式允许：

\[
$\eta_{k,l}^{service}\neq \eta_l^{power}$\]

从而避免把功率缩放与吞吐缩放错误地设为同一比例，也能反映内存墙、通信瓶颈和不同任务类型的响应差异。虽然该模型包含二进制变量，但仍属于线性 MILP。

---

# 16. 可调节范围计算

柔性负荷能力应通过整个可行域在功率轴上的投影计算。

对每个时段 \(t\)，在固定此前已实现状态的条件下，分别求解：

\[
$P_t^{\max}=\max P_t^{DC}\tag{57}$\]

\[
$P_t^{\min}=\min P_t^{DC}\tag{58}$
\]

两个问题均满足原模型全部队列、截止时间、连续运行、光纤、资源容量、供电和动态约束。

全可行功率区间为：

\[
$P_t^{DC}\in\left [P_t^{\min}, P_t^{\max}\right]\tag{59}$
\]

总可调宽度为：

\[
$\Delta P_t^{flex}= P_t^{\max}-P_t^{\min}\tag{60}$
\]

若基准经济最优运行点为 \($P_t^{*}$\)，则该运行点的上下调能力为：

\[
$F_t^{up}= P_t^{\max}-P_t^{*}\tag{61}$
\]

\[
$F_t^{down}= P_t^{*}-P_t^{\min}\tag{62}$
\]

应同时报告全可行区间与基准运行点调节能力，不能只给出单一“可调容量”。

# 17. 模型输出指标

模型输出以下逐时结果：

1. 各类任务完成量 \($x_{k,t}$\)；
2. 可选任务完成量 \($x_t^s$\)；
3. 任务积压量 \($q_{k,t}$\)；
4. 逾期任务量 \($h_{k,t}$\)；
5. 连续块任务运行状态 \($y_{j,t}$\)；
6. IT 功率 \($P_t^{IT}$\)；
7. 辅助功率 \($P_t^{aux}$\)；
8. 数据中心总功率 \($P_t^{DC}$\)；
9. 绿色供电和稳定供电功率；
10. 光纤数据流量和缓存状态；
11. DVFS 运行点；
12. 时段可行功率上下界。

总计算服务量：

\[
$X^{total}=\sum_{t\in\mathcal T}\left(\sum_{k\in\mathcal K}x_{k,t}+x_t^s\right)+\sum_{j\in\mathcal J^b}W_ja_j\tag{63}$
\]

数据中心总耗电量：

\[
$E^{DC}=\sum_{t\in\mathcal T}P_t^{DC}\Delta t\tag{64}$
\]

绿色电力占比：

\[
$\eta^{green}=\frac{ \sum_tP_t^{green}\Delta t}{\sum_tP_t^{DC}\Delta t}\tag{65}$
\]

合同任务完成率：

\[
$\eta^{service}=\frac{\sum_{t,k}x_{k,t}}{\sum_{t,k}A_{k,t}}\tag{66}$
\]

平均队列等待量：

\[
$\bar Q=\frac{1}{T}\sum_{t,k}q_{k,t}\tag{67}$
\]

算力负荷率：

\[
$LF^{IT}=\frac{ \sum_tP_t^{IT}\Delta t}{C^{IT}T\Delta t}\tag{68}$
\]
