# 4.9 优化调度模块

当前V4纳入的是可执行的跨模块可行性调度器，不宣称已经完成Pareto最优、随机优化或滚动优化。

- `model/v4_dispatch_4_9.m`：读取4.3供给边界与4.6算力可行边界，生成4.5、4.6、4.7请求及计划弃电/缺供。
- `model/v4_validate_dispatch_4_9.m`：检查容量、SOC、充放互斥和计划功率闭合。
- `model/v4_reconcile_actuals_4_9.m`：根据4.3—4.7实际响应确认实际弃电和分类缺供，再交4.4核账。
- `docs/4.9调度与求解说明.md`：说明本版能力边界和升级接口。

设备效率、SOC、库存和交付能力仍由4.3—4.7各责任模块执行，4.9不得改写其物理状态。
