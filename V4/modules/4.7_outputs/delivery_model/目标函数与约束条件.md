# 目标函数与约束条件

## 1. 符号

| 符号 | 含义 | 单位 |
|---|---|---|
| `P_available,t` | t 时刻能源岛可用电力 | MW |
| `P_grid,t` | t 时刻海缆送端外送功率 | MW |
| `P_grid_del,t` | t 时刻陆上实际接收功率 | MW |
| `P_compute,t` | t 时刻算力中心设施总用电功率 | MW |
| `P_it,t` | t 时刻 IT 设备用电功率 | MW |
| `X_compute,t` | t 时刻完成的算力服务量 | MWh-IT |
| `P_h2_el,t` | t 时刻电解制氢功率 | MW |
| `H_prod,t` | t 时刻制氢量 | kg |
| `H_pipe,t` | t 时刻管道输出氢气量 | kg |
| `H_ship,t` | t 时刻船运输出氢气量 | kg |
| `H_storage,t` | t 时刻储氢量 | kg |
| `P_marine,t` | t 时刻海洋综合用能功率 | MW |
| `P_curt,t` | t 时刻弃电功率 | MW |
| `Delta_t` | 时间步长 | h |

## 2. 目标函数

当前代码实现的是运行毛利，不含完整 CAPEX 年化成本：

```text
Max OperatingMargin
= sum_t (
    R_power,t
  + R_compute,t
  + R_h2,t
  + V_marine,t
  - C_power_var,t
  - C_compute_var,t
  - C_h2_var,t
  - Penalty_marine_unserved,t
)
```

### 电力收益

```text
R_power,t = P_grid_del,t * Delta_t * 1000 * price_power
```

其中 `price_power` 单位为元/kWh。

### 算力收益

```text
R_compute,t = X_compute,t * price_compute
```

其中 `price_compute` 单位为元/MWh-IT。

### 氢气收益

```text
R_h2,t = H_delivered,t * price_h2
```

其中 `H_delivered,t` 为扣除管道或船运损耗后的实际交付氢气量。

## 3. 电力-海缆约束

```text
0 <= P_grid,t <= min(P_grid_max, P_grid_accept_max)
```

```text
P_grid_del,t = P_grid,t * (1 - loss_cable)
```

当前代码采用线性损耗。后续可升级为随距离和负载率变化的二次损耗模型。

## 4. 算力-光缆约束

算力设施功率约束：

```text
P_compute,t = 0
or
P_compute_min <= P_compute,t <= P_compute_max
```

PUE 关系：

```text
P_compute,t = P_it,t * PUE
```

算力服务量：

```text
X_compute,t = P_it,t * Delta_t
```

光缆服务约束：

```text
0 <= X_compute,t <= B_fiber_service_max * Delta_t
```

注意：光缆传输的是数据和算力服务结果，不是电能。因此光缆不进入电力损耗模型。

## 5. 氢气-管道/船运约束

电解槽功率约束：

```text
0 <= P_h2_el,t <= P_h2_el_max
```

制氢量：

```text
H_prod,t = 1000 * P_h2_el,t * Delta_t / SEC_H2
```

其中 `SEC_H2` 单位为 kWh/kg。

输氢能力约束：

```text
0 <= H_pipe,t <= H_pipe_max * Delta_t
0 <= H_ship,t <= H_ship_max * Delta_t
```

储氢状态：

```text
H_storage,t+1 = H_storage,t + H_prod,t - H_pipe,t - H_ship,t
```

储氢容量：

```text
0 <= H_storage,t <= H_storage_max
```

## 6. 海洋综合用能约束

```text
0 <= P_marine_served,t <= P_marine_request,t
```

如果海洋综合用能中存在刚性负荷，则应满足：

```text
P_marine_served,t >= P_marine_rigid,t
```

若无法满足，代码会记录未供电惩罚。

## 7. 综合功率平衡

当前代码实现的综合平衡为：

```text
P_available,t + P_storage_dis,t
=
P_grid,t
+ P_compute,t
+ P_h2_el,t
+ P_marine,t
+ P_storage_ch,t
+ P_curt,t
```

当前版本没有实现电池储能子模型，因此 `P_storage_ch,t` 和 `P_storage_dis,t` 仅作为接口保留。

## 8. 非负约束

所有物理流量均非负：

```text
P_grid,t >= 0
P_compute,t >= 0
P_h2_el,t >= 0
P_marine,t >= 0
P_curt,t >= 0
H_pipe,t >= 0
H_ship,t >= 0
H_storage,t >= 0
```

## 9. 当前代码覆盖关系

| 文件 | 对应模型 |
|---|---|
| `power_export.py` | 电力-海缆外送模型 |
| `compute_load.py` | 算力-光缆输出模型 |
| `hydrogen_output.py` | 氢气-管道/船运输出模型 |
| `marine_load.py` | 海洋综合用能模型 |
| `balance.py` | 综合功率平衡与分账 |
| `objectives.py` | 目标函数收益/成本汇总 |
| `constraints.py` | 目标函数与约束条件清单 |
