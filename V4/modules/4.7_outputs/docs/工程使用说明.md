# 蓝海枢纽4.7用能与外送子模型工程使用说明

## 工程作用

本工程用于建立“蓝海枢纽”项目中电力外送、算力服务、氢能交付和海洋综合用能的可运行子模型。当前版本是“可审计子模型原型”，用于参数校准、典型日测算、约束检查及接入4.9优化器前的公式验证，不是完整优化求解器。

当前代码覆盖：

- 电力—海底电缆外送：海缆容量、登陆点接纳、输送损耗、电力收入和外送约束。
- 算力—光缆输出：设施用电、PUE转换后的IT服务量、光缆交付能力和算力收入。
- 氢能输出：接收4.5提供的制氢量与储氢状态，计算管道／船运能力、损耗、交付量和收入。
- 海洋综合用能：岛上辅助负荷、海水淡化、海洋装备用能、缺供惩罚和服务量。
- 综合功率平衡：把电力外送、算力、制氢用电和海洋负荷接入统一时步的调度检查框架。

> 边界提醒：4.5负责电池、制氢和储氢内部状态；4.7只接收其接口输出并处理交付通道，不重复建立储能或电解槽调度。

## 参数配置

参数集中放在`submodules/configs/parameters.yaml`。企业参数到手后优先修改配置，不直接改公式代码。

- `power_export`：海缆容量、登陆点接纳、损耗、电价、变动成本、投资和运维率。
- `compute`：IT容量、最小负荷、PUE、光缆能力、带宽、延迟、服务价格和成本。
- `hydrogen`：氢气交付接口、管道／船运能力、损耗率、氢价和运输成本。
- `marine`：辅助负荷、淡化负荷、装备负荷、柔性比例、服务价值和缺供惩罚。

## 环境与运行

建议使用Python 3.11或更高版本和uv。在`submodules`目录运行：

```powershell
$env:UV_CACHE_DIR = Join-Path $PWD '.uv-cache'
uv run python -m unittest discover -s tests
uv run python examples/run_example.py
```

测试通过应显示`Ran 17 tests`和`OK`。示例生成24小时典型日的外送电量、到岸电量、算力服务量、氢气交付量、海洋负荷服务量、弃电、收入、成本、经营毛利、功率平衡残差和约束违背次数。

## 当前模型边界

- 不做详细电网潮流，只做海缆容量、登陆接纳和损耗约束。
- 不做通信网络路由，光缆仅作为算力服务交付能力约束。
- 不做船舶逐航次调度，船运暂用等效`kg/h`能力表示。
- 不重复模拟4.5内部电池SOC、电解槽状态和储氢库存。
- 不做完整财务模型，当前输出以运行收入、变动成本和经营毛利为主。
- 示例采用简化策略；最优容量和运行由4.9统一优化模型处理。

## 企业数据补充顺序

1. 登陆点逐时接纳能力、海缆容量和损耗口径。
2. 电价、算力服务价格、氢气交付价格及合同边界。
3. IT容量、PUE、可迁移任务比例、光缆带宽和可用率。
4. 从4.5接收的制氢量、储氢可用量、产品压力／纯度及交付边界。
5. 管道／船运能力、物流损耗、海洋用能需求和缺供代价。
